# 2026-04-27 — Decision: scripted demos exclusively, Mimic is out

## Decision
Going forward, the VLA cube-pick training dataset is generated by
SCRIPTED demos only. Isaac Lab Mimic data generation is no longer part of
the pipeline.

## Why this is in writing
The pipeline change touches Phase 2 of the recovery plan and any future
training-run plan. Without an explicit decision record, future-me would
read the Mimic-flavored recovery plan and waste time re-validating the
abandoned path. This file is the audit trail.

- `reports/recovery_plan_2026-04-24.md` — Phase 2 rewritten in place; §1
  executive-summary line updated; cost table updated. A 2026-04-27
  banner at the top points readers here.
- `reports/2026-04-26_scene_data_integrity_pack.md` — forward-looking
  Mimic mentions in the "Next step" section replaced with scripted-only

- bd issues — `vla_kitting-mil` description had "(mimic + scripted demos)"
  → now "scripted-demo training labels". `vla_kitting-8ux` (v5 training)

  still imported when older scripts touch the registered task ID. No
  cleanup needed unless we find a concrete reason.
- Closed bd issues that reference Mimic (e.g. vla_kitting-966 close
  notes describing the per-episode color path through Mimic) are
  historical and untouched.
- The 2026-04-26 30-demo scripted dataset under
  `/tmp/yaw_30/cube_scripted_yaw30.hdf5` is unchanged — it was already
  scripted-only.

## Implications for v5 (`vla_kitting-8ux`)
Plan, in order:
1. Decide xy_tolerance (vla_kitting-mil) — still gating.
2. Generate scripted-only training dataset by running
   `scripts/validate/scripted_pick_demo.py` at scale (target N
   somewhere in [400, 750] depending on how much wall time we want to
   spend; Run B prime trained on ~750 Mimic demos so 750 scripted is
   the apples-to-apples comparison).
3. Convert via `isaaclab_to_lerobot.py --drop_cube_pos --stride 1`.
   The converter already reads `attrs[task]` for per-episode prompts,
   so per-episode color randomization carries through.
4. Train under the Run B prime LoRA recipe (r=16, alpha=8, no vision
   LoRA, no `modules_to_save`, uniform L2, batch 4) per
   reports/2026-04-26_run_b_prime_config_diff.md.
5. Eval on `configs/eval_seed_30.json` for the apples-to-apples SR
   number. (Re-bake first if cube spawn box has changed since 2026-04-26;
   the spawn box was rotated and re-centered, and a new corner-coverage
   bd issue vla_kitting-v67 is in flight.)

## Risk register for the scripted-only path
| Risk | Mitigation |
|---|---|
| Scripted demos all use the same 12-phase trajectory shape, so action diversity per cube position may be lower than Mimic's spliced trajectories | Hold to the 2026-04-26 finding (`scripted_action_variance.py`): ee_dx/ee_dy ARE diverse at gain=2. If v5 still 0/N, run that diagnostic again on the new scaled dataset before assuming data is fine. |
| Scripted controller wallclock × N demos may be the bottleneck (~30 s/demo serial, no Mimic parallelism) | Parallelize across multiple env instances (num_envs > 1) once we confirm the scripted controller works under that, OR accept the 6-15 hr generation cost as a one-time. |
| Without Mimic, dataset diversity is 1:1 with `randomize_cube_color`, `randomize_pose`, `randomize_dome_light_intensity` event variety. If those random distributions undersample some region, the policy will silently undertrain there | Sanity check at Phase 2 gate G2: cube-position spread + per-color count, both already required. |
| Run B prime trained on Mimic data; comparing v5 (scripted) to it confounds LoRA-recipe with data-source | Plan accordingly: if v5 underperforms Run B's 10% SR, we can't blame LoRA alone; we'd have to ablate dataset source. The "apples-to-apples" framing in 8ux is honest about this. |

## Lesson
The "Mimic for sample efficiency" assumption was inherited from the
Franka reference example. We don't have evidence that Mimic actually
helps for this task on this rig — Run B prime was the only successful
checkpoint and even it hit only 10% SR, so Mimic-trained ≠
working. Cutting it removes a layer of pipeline complexity (two
different recorder paths, the Mimic env cfg, the annotate_demos
wrapper) without giving up anything we've measured.
